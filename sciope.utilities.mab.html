sciope.utilities.mab.mab_direct module¶

Multi-Armed Bandits: Direct Algorithm Solution Class

class sciope.utilities.mab.mab_direct.MABDirect(arm_pull, epsilon=1, delta=0.4, use_logger=False)[source]¶

Bases: sciope.utilities.mab.mab_base.MABBase

The DIRECT algorithm pulls each arm a fixed number of times such that with high probability (1-delta), the k selected arms with highest empirical averages are all (epsilon, k)-optimal. Ref: “Efficient Selection of Multiple Bandit Arms: Theory and Practice”, Shivaram Kalyanakrishnan and Peter Stone

sciope.utilities.mab.mab_halving module¶

Multi-Armed Bandits: Halving Algorithm Solution Class

class sciope.utilities.mab.mab_halving.MABHalving(arm_pull, epsilon=1, delta=0.4, use_logger=False)[source]¶

Bases: sciope.utilities.mab.mab_base.MABBase

Halving improves over the incremental method by eliminating half of the arms from round l to the round l+1. Arms are sampled enough times in each round to ensure that at least k(epsilon, k)-optimal arms are likely to survive each round. Ref: “Efficient Selection of Multiple Bandit Arms: Theory and Practice”, Shivaram Kalyanakrishnan and Peter Stone

sciope.utilities.mab.mab_incremental module¶

Multi-Armed Bandits: Incremental Algorithm Solution Class

class sciope.utilities.mab.mab_incremental.MABIncremental(arm_pull, epsilon=1, delta=0.4, use_logger=False)[source]¶

Bases: sciope.utilities.mab.mab_base.MABBase

The incremental algorithm consists of ‘k’ rounds. In a given round l, S_l is the set of selected arms, while R_l is the set of arms remaining to be processed. During round l, an (epsilon,1) optimal arm in R_l is selected with high probability using the median elimination algorithm (Even-Dar et. al. 2006) Ref: “Efficient Selection of Multiple Bandit Arms: Theory and Practice”, Shivaram Kalyanakrishnan and Peter Stone

sciope.utilities.mab.mab_sar module¶

Multi-Armed Bandits: Successive Accepts and Rejects (SAR) Algorithm Solution Class

class sciope.utilities.mab.mab_sar.MABSAR(arm_pull, p, b, use_logger=False)[source]¶

Bases: sciope.utilities.mab.mab_base.MABBase

SAR divides n rounds into p-1 phases. At the end of each phase, the algorithm either accepts the arm with the highest empirical mean reward, or rejects the arm with the lowest empirical mean reward. The arm in question is deactivated. In the next phase, each active arm is pulled equally often. Ref: “Multiple Identifications in Multiple-Armed Bandits”, Bubeck, Wang and Vishwanathan (2012)
